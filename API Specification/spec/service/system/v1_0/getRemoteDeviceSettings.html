system - v1.0 None

getRemoteDeviceSettings Â¶

This API provides current settings and supported settings related to remote device, which can access server device from outside the door.
Syntax
http://<Base URL>/system
ws://<Base URL>/system
Authentication Level
none
  • Request
  • Response
  • Notification
  • See also
params' Elements
An object composed by following pair(s).
name type multiplicity default description
target string ? ""
Target name. Default value is "". It means settings of all targets. Client can get all remote device settings information by setting empty string to "target". Client can get specific remote device setting value by explicitly setting target name to "target".
  • "accessPermission" - Set whether to permit access from remote device, which can access server device from outside the door. (more...)
  • "" - Settings of all targets.
JSON Example
{
    "id": 44,
    "method": "getRemoteDeviceSettings",
    "params": [{"target": "accessPermission"}],
    "version": "1.0"
}
result's Elements
An array of objects composed by following pairs.
name type multiplicity default description
target string 1  
Target name.
  • "accessPermission" - Set whether to permit access from remote device, which can access server device from outside the door. (more...)
currentValue string 1  
Current value of target.
  • In case "target" is "accessPermission"  (more...)
    • "on" - access permitted.
    • "off" - access not permitted.
deviceUIInfo string ? ""
This parameter provides information how devices display UI information. This format is product dependent and actual format should be explained in Device Specific Info page .
title string ? ""
Title of settings for display purpose. "" means no specified title.
titleTextID string ? ""
The identifier, which is unique in product, assigned by server to identify title. "" means no specified title text ID.
type string ? ""
Type of value to identify actual value type of target.
  • "enumTarget" - Enumeration type for specifying finite candidates. (ex) "high", "mid", "low"
  • "booleanTarget" - Boolean type only two candidates available. (ex) "off, "on"/"false", "true"
  • "integerTarget" - Integer type. (ex) "1", "-10"
  • "doubleNumberTarget" - Number type including double defined in Scalar Web API. (ex) "1.5", "-10.0"
  • "stringTarget" - String type. (ex) "hello"
  • "" - no type information.
isAvailable boolean ? true
This target is currently available or not.
candidate (object-array) ? null
Candidates of specified settings. Only If there is no candidate, null is set.
  title string ? ""
Title of candidate for UI display purpose. "" means no specified title
  titleTextID string ? ""
The unique identifier, which is unique in product, assigned by server to identify title. "" means no specified title text ID.
  value string ? ""
Value of candidate. "" can be set only when settings value is integer and the candidate can be represented by "max", "min", and "step".
  isAvailable boolean ? true
This candidate is currently available or not.
  max double ? -1
Max value of specified settings. Only If value of target is not indicated by numerical number, -1 is set.
  min double ? -1
Min value of specified settings. Only If value of target is not indicated by numerical number, -1 is set.
  step double ? -1
Step value of specified settings. Only If value of target is not indicated by numerical number, -1 is set.
JSON Example
{
    "id": 44,
    "result": [[{
        "target": "accessPermission",
        "candidate": [
            {
                "title": "On",
                "value": "on",
                "isAvailable": true
            },
            {
                "title": "Off",
                "value": "off",
                "isAvailable": true
            }
        ],
        "currentValue": "on",
        "isAvailable": true
    }]]
}

No additional error code is defined. Refer to error code for common errors.
