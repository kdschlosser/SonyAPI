videoScreen - v1.0 None

requestToNotifyScreenState Â¶

This API provides a function to notify screen state change of device. This API returns response when at least one of specified request parameters is different from current device status.
Syntax
http://<Base URL>/videoScreen
ws://<Base URL>/videoScreen
Authentication Level
none
  • Request
  • Response
  • Notification
  • See also
params' Elements
An object composed by following pair(s).
name type multiplicity default description
multiScreenMode string 1  
screen mode of device
  • "single" - single screen mode
  • "PIP" - Picture In Picture mode
  • "PAP" - Picture And Picture mode
  • "widgetMode" - Picture and Widget application mode
  • "internetPhoneMode" - Internet Phone application mode
  • "other" - other mode
pipSubScreenPosition string 1  
sub screen position as below.
  • "leftTop" - shown in the upper left corner of main screen
  • "leftBottom" - shown in the lower left corner of main screen
  • "rightTop" - shown in the upper right corner of main screen
  • "rightBottom" - shown in the lower right corner of main screen
  • "invalid" - sub screen is not shown. That means the current Multi Screen Mode is not PIP.
audioSourceScreen string 1  
video screen with sound output.
  • "main" - main screen
  • "sub" - sub screen
option (object) ? null
optional information.
  internetTVMode boolean ? false
Indicates whether the current screen mode is customized for Internet TV.
  • true - the screen mode is for Internet TV device
  • false - otherwise
JSON Example
{
    "id": 30,
    "method": "requestToNotifyScreenState",
    "params": [{
        "pipSubScreenPosition": "leftTop",
        "multiScreenMode": "PIP",
        "audioSourceScreen": "sub",
        "option": {"internetTVMode": true}
    }],
    "version": "1.0"
}
