avContent - v1.2 Private

getContentList Â¶

This API provides the list of contents under the scheme and source. For example, on the device supporting broadcast contents like television device, this API returns broadcasting channels by indicating "tv:*" as source. And this API returns list of recorded contents of TV program in case of "usb:recStorage" as source.

In case the number of contents is too large to be retrieved in a single request, 'stIdx' and 'cnt' parameters should be used to retrieve partial lists. To get the complete list, multiple requests are to be made adjusting the 'stIdx' and 'cnt' parameters. There is a maximum limit on the number of contents that can be retrieved in a single request. This limit is device specific. The 'cnt' parameter also has the same maximum limit.
Syntax
http://<Base URL>/avContent
ws://<Base URL>/avContent
Authentication Level
private

  • Response

params' Elements
An object composed by following pair(s).
name type multiplicity default description
source string 1  
Source name that can be retrieved by v1_0.GetSourceList.
stIdx integer ? 0
Start index to get list items.
cnt integer ? 50
Count of the maximum number of items that can be listed, starting from "stIdx".
type string ? ""
Optional parameter to narrow down result within specified source in the request. Following values are defined.
  • "general" - This parameter is only for "tv:dvbs" as source parameter. Returned list contains all information.
  • "preferred" - This parameter is only for "tv:dvbs" as source parameter. Returned list only contains the information of Satellite Operator set by user in the device.
  • "" - Not specified.
target string ? ""
Optional parameter to widen result within specified source in the request. Following values are defined.
  • "all" - This parameter is currently only for "tv:isdbt" / "tv:isdbbs" / "tv:isdbcs" / "tv:isdbgt" / "tv:antenna" / "tv:cable". Returned list also contains the contents with the value "invisible" in epgVisibility, channelSurfingVisibility or channelSurfingVisibility.
  • "" - Not specified.
JSON Example
{
    "id": 2,
    "method": "getContentList",
    "params": [{
        "cnt": 100,
        "source": "tv:dvbs",
        "stIdx": 10,
        "type": "general"
    }],
    "version": "1.2"
}

{
    "id": 2,
    "method": "getContentList",
    "params": [{
        "cnt": 100,
        "source": "tv:isdbt",
        "stIdx": 0,
        "target": "all"
    }],
    "version": "1.2"
}

{
    "id": 2,
    "method": "getContentList",
    "params": [{"source": "video:content?productID=***"}],
    "version": "1.2"
}
result's Elements
An array of objects composed by following pairs.
name type multiplicity default description
uri string 1  
URI to identify the content. Refer to here to know URI structure in detail.
  • (ex) "tv:isdbt?trip=11.22.33"
title string ? ""
Title of this content to be recognized by user.
  • (ex) "Sports Channel"
  • (ex) "NHK"
index integer ? 0
Index of the list. This starts with "stIdx" indicated in the request.
dispNum string ? ""
Numeric string of the channel to be recognized by user. Default value is "" and means undefined.
  • (ex) "101"
  • (ex) "211.2"
originalDispNum string ? ""
Original display number in broadcasting data table. Default value is "" and means undefined.
tripletStr string ? ""
Triplet string to identify channel. Default value is "" and means undefined.
  • (ex) "11.22.33"
programNum integer ? -1
Program number on broadcasting data table. Default value is -1 and means undefined.
programMediaType string ? ""
Media type of broadcast program.
  • "tv" - means TV program
  • "radio" - means radio program
  • "data" - means data program
  • "" - unknown type
directRemoteNum integer ? -1
Mapped number button on remote controller. Default value is -1 and means undefined. (usually 1-12)
name type multiplicity default description
epgVisibility string ? "auto"
Setting of visibility on EPG application.
  • "visible" - This content is shown in EPG.
  • "invisible" - This content is hidden in EPG.
  • "auto" - This content is shown if this content is main channel in EPG.
channelSurfingVisibility string ? "visible"
Setting of visibility on pushing <Channel+> / <Channel-> button.
  • "visible" - This content is shown.
  • "invisible" - This content is hidden.
name type multiplicity default description
visibility string ? "visible"
Setting of visibility. This is general setting and should be exclusive with epgVisibility and channelSurfingVisibility.
  • "visible" - This content is shown.
  • "invisible" - This content is hidden.
name type multiplicity default description
uri string 1  
URI to identify the content. Refer to here to know URI structure in detail.
  • (ex) "usb:recStorage?path=%2FRec%2FUSB%2Fusb1%2F771%2F00000001.cdb"
title string ? ""
Title of this content to be recognized by user.
  • (ex) "News24"
  • (ex) "Baseball Game"
index integer ? 0
Index of the list. This starts with "stIdx" indicated in the request.
startDateTime string ? ""
Scheduled date and time to start. The format is ISO-8601 as described below. Default value is "" and means undefined. (YYYY-MM-DDTHH:MM:SS.sTZD)
channelName string ? ""
Broadcaster channel name of the recorded content. Default value is "" and means undefined.
fileSizeByte integer ? -1
File size of the content. Default value is -1 and means undefined.
isProtected boolean ? false
This value represents whether content is protected against being deleted.
  • true - protected
  • false - not protected
isAlreadyPlayed boolean ? false
This value represents whether the content have even been played or not by user.
  • true - already played
  • false - not played yet
durationSec double ? -1
Duration in seconds. Default value is -1 and means undefined.
name type multiplicity default description
uri string 1  
URI to identify the content. Refer to here to know URI structure in detail.
  • (ex) "extInput:hdmi?port=1"
title string ? ""
Title of this content to be recognized by user.
index integer ? 0
Index of the list. This starts with "stIdx" indicated in the request.
