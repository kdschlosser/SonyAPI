audio - v1.0 Generic

setSpeakerSettings Â¶

This API provides a function to change settings related to speaker setting items.
Syntax
http://<Base URL>/audio
ws://<Base URL>/audio
Authentication Level
generic
  • Request
  • Response
  • Notification
  • See also
params' Elements
An object composed by following pair(s).
name type multiplicity default description
settings (object-array) 1  
Object to accommodate one or more target/value pair.
  target string 1  
Target name. (UI setting target)
  • "inCeilingSpeakerMode" - Set whether or not to use in-ceiling speaker mode with the current input. In a setup where the front and center speakers are installed in the ceiling, lowering the audio output position to the screen height allows you to enjoy a more natural experience. (more...)
  • "tvPosition" - Setting the sound according to the TV position. (more...)
  • "speakerSelection" - Select the front speaker system to operate. (more...)
  • "frontLLevel" - Set level of Front Left speaker. (more...)
  • "frontRLevel" - Set level of Front Right speaker. (more...)
  • "centerLevel" - Set level of Center speaker. (more...)
  • "surroundLevel" - Set level of Surround speaker. (more...)
  • "surroundLLevel" - Set level of Surround Left speaker. (more...)

  • "frontHighRLevel" - Set level of Front High Right speaker. (more...)
  • "subwooferLevel" - Set level of Subwoofer speaker. Note that the range and step values vary depending on device. (more...)
  • "subwooferFreq" - Adjust the cut off frequency of the Wireless Subwoofer. All frequencies below the cut off frequency are output to the Wireless Subwoofer instead of the TV speakers. (more...)
  • "subwooferPhase" - Sets the phase polarity of subwoofer. (more...)
  • "subwooferPower" - Set the power control method of the Wireless Subwoofer. (more...)

Error Code
Following error codes are important for this API. Refer to error code for other errors.
code Explanation
40004
One or more settings are not set by error when multiple settings are set. Client needs to call paired APIs (getXXXSettings) to identify which parameters fail to be updated.
