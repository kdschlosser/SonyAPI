recording - v1.0 None

getRecordingStatus Â¶

This API provides Recording Status of the device. Recording Status here indicates the status of the entire schedule list. For example if any one schedule is under recording, it returns "recording" as result.
Syntax
http://<Base URL>/recording
ws://<Base URL>/recording
Authentication Level
none
  • Request
  • Response
  • Notification
  • See also
params' Elements
Not necessary.
JSON Example
{
    "id": 50,
    "method": "getRecordingStatus",
    "params": [],
    "version": "1.0"
}
result's Elements
An object composed by following pair(s).
name type multiplicity default description
status string 1  
Recording Status
JSON Example
{
    "id": 50,
    "result": [{"status": "recording"}]
}
