recording - v1.0 Generic

addSchedule Â¶

This API provides function to register new scheduled item for Recording/Reminder.

In case registration fails, it returns an error code as result. And in case the registration succeeds with some warning, it returns an annotation code as result. The warnings mean that the registered item may not be executed as specified in the request. For example, in the case of returned annotation: 3 (Storage is full), the item is registered successfully but it might fail unless storage capacity is enough at the time of execution.
Syntax
http://<Base URL>/recording
ws://<Base URL>/recording
Authentication Level
generic
  • Request
  • Response
  • Notification
  • See also
params' Elements
An object composed by following pair(s).
name type multiplicity default description
type string ? "recording"
Type of schedule
uri string 1  
URI to identify channel.
  • (ex) "tv:isdbt?trip=11.22.33"
title string ? ""
Name of scheduled item. Default value is "" and actual title to be set by device automatically.
startDateTime string 1  
Scheduled date and time to start. The format is ISO-8601 as described below. (YYYY-MM-DDTHH:MM:SS.sTZD)
durationSec integer ? 0
Duration in seconds. This parameter can be optional only when type is "reminder". In such case, default value is 0 which means there is no specific end time for "reminder".
repeatType string ? "1"
Repeat type

Supported repeatType values can be retrieved by using getSupportedRepeatType.
quality string ? "DR"
Recording mode for video quality. In case type is "reminder", this parameter is ignored.
  • "DR" - Direct recording
JSON Example
{
    "id": 33,
    "method": "addSchedule",
    "params": [{
        "startDateTime": "2012-10-13T22:00:00+0000",
        "durationSec": 3600,
        "type": "recording",
        "uri": "tv:isdbt?trip=11.22.33"
    }],
    "version": "1.0"
}
result's Elements
An object composed by following pair(s).
name type multiplicity default description
annotation integer ? 0
Annotation code that indicates the result of the request.
  • 0 - No annotation
  • 1 - Storage (HDD or Disc) not ready
  • 2 - Additional storage not registered
  • 3 - Storage is full
  • 4 - Storage is almost full
  • 5 - The number of titles in storage is maximum
JSON Example
{
    "id": 33,
    "result": [{"annotation": 2}]
}
