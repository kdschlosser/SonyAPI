Device Resource URI Â¶

In Scalar Web API, standard URI structure, as defined by RFC 3986, is used for representing device's resources. Schemes are used to refer to device resources. URI is basically provided from devices/servers, and a client should not change/modify the URI.

Similar URIs can be used with different services. For example, TV's channel URI can be used with avContent and recording.

As to the supported URIs in a server device, see Supported URIs in each device specific information (Server Device Specific information).

Examples
Notice that every source of "tv" scheme refers to digital resource. Analog resource is not supported in this API version. For example, tuner resources for DVB region are shown as
  • tv:dvbt?trip=11.22.33&srvName=ABC
  • tv:dvbc?trip=11.22.33&srvName=ABC
  • tv:dvbs?trip=111.222.333&srvName=ABC
For example, tuner resources for ISDB region are shown as
  • tv:isdbt?trip=111.222.333
  • tv:isdbbs?trip=111.222.333
  • tv:isdbcs?trip=111.222.333
For example, tuner resources for ATSC region are shown as
  • tv:antenna?dispNum=111.222
  • tv:cable?dispNum=111.222
  • (From TV device interface 2.X.Y) tv:antenna?dispNum=111.222&index=0
  • (From TV device interface 2.X.Y) tv:cable?dispNum=111.222&index=3
      • (Note: If index is not specified, uri should be handled as invalid)
For example, tuner resource for ISDBG region is shown as
  • tv:isdbgt?dispNum=111.222
For example, tuner resource for SkyPerfectTV is shown as
  • tv:dvbsj?trip=11.22.33&srvName={CHANNELNAME}
For example, TV contents managing favorite resources (4 lists is managed.) are shown as
