Error Code ¶

In Scalar Web API Protocol, HTTP and JSON-RPC are used, so we can use status code of both layers. This document describes about error handling in different layers and lists examples for major errors.
Overview
Error code is defined between 0-65535. The error code is divided into two areas (Scalar Web API system area and user area).
error code area
0 - 32767 Scalar Web API system area
32768 - 65535 user area
User area is divided for common area and each service.
error code area
32768 - 39999 unused area
40000 - 40199 service common area
40200 - 40399 system service area
40400 - 40599 camera service area
40600 - 40799 videoScreen service area
40800 - 40999 audio service area
41000 - 41199 avContent service area
41200 - 41399 recording service area
41400 - 41599 appControl service area
41600 - 41799 browser service area
42000 - 42199 accessControl service
42200 - 42399 contentDownload service
42400 - 42599 encryption service
42600 - 42799 contentSync service
42800 - 42999 contentSync service
43000 - 65535 reserved area

HTTP Major Error Code in Scalar Web API
This is basic policy.
  • Should follow original HTTP Definition
  • Should not map others to HTTP error forcibly
In case HTTP status code is other than 200 OK and if HTTP server returns response body in accordance with Scalar Web API protocol, error code must be same as HTTP error code. Here, major error codes are listed up.
code Reason phrase example JSON example Explanation
401 Unauthorized "error": [401, "Unauthorized"] The request requires user authentication.
403 Forbidden "error": [403, "Forbidden"] The server understood the request, but is refusing to fulfill it. Client does not have permission to access.
404 Not Found "error": [404, "Not Found"] For cases where request is not matched to any supported API version.
413 Request Entity Too Large "error": [413, "Request Entity Too Large"] The accepted body size of a client request exceeds maximum.
414 Request-URI Too Long "error": [414, "Request-URI Too Long"] The accepted URI length of a client request exceeds maximum.
501 Not Implemented "error": [501, "Not Implemented"] When the request method is not implemented on the server.
503 Service Unavailable "error": [503, "Service Unavailable"] When server is in temporally unavailable state which may occur due to over concurrent connections. (We don't define the number of max connections because it depends on servers.)
