Introduction+

This page contains additional information related to implementation of Scalar Web API on TV for version 2. If you start development of Client or Server side applications that use Web APIs on TV, this will help you to understand its behavior and restriction in detail.

Device Discovery+

TV device's DD conforms to Format A.

Access Control+

All APIs except photoshare service conform to default authentication levels. actRegister API on accessControl service and web application authentication can be used.

Supported Protocol+

HTTP is supported.

getSystemInformation v1.0+

Guide about Response+

  • TV device returns fixed values for following parameters.

    namevalue
    product"TV"
    name"BRAVIA"
  • "region" and "area" store same value. TV extends ISO-3166-1 and uses original codes.
  • TV extends ISO-639 alpha-3 also on "language".
  • "macAddr" stores valid mac address of active NIC. The format must be as "xx:xx:xx:xx:xx:xx".  In TV device, only one NIC is active.

  • "generation" stores same value as interfaceVersion of getInterfaceInformation.

Guide about Response+
  • When the value of option parameter is "WOL", the value of value parameter stores MAC address about active NIC. In TV device,  only one NIC is active. If this API is called via wireless NIC, value stores MAC address of wireless NIC.
  • "SupportedChineseSoftwareKeyboard" is deprecated. Please don't use.

setLanguage v1.0+

The available values of "language" depend on region or country settings on TV. If you send a request with an unsupported value for "language", error code will be returned. For example, if you send a request with "fre" value to JP set, it will be denied.
Special values out of ISO-639 alpha-3 are defined as below;

  • "CHS": means Simplified Chinese. It's available only in DVB-CN model .
  • "CHT": means Traditional Chinese. It's available only in DVB-TW model .

 

getCurrentTime v1.1+

In UC region set, the values of " timeZoneOffsetMinute" and "dstOffsetMinute" may not be reliable. TimeZone settings in this region's TV can't be set by user and cannot be changed automatically, so it keeps default value even if Country or ZIP code setting is changed.

setCurrentTime v1.0+

In case that the current date and time in TV is already set by TV broadcasting or internet clock, this API cannot change it and returns error code instead.
Note if set "timeZoneOffsetMinute" and "dstOffsetMinute" values in a request are set,  these values oare only useful in DVB region. In other regions, for example JP or US, these values will be ignored. Besides as for "dateTime" value, set UTC based date and time in that case.

getPostalCode v1.0 / setPostalCode v1.0+


The format of "postalCode" depends on region and country settings.

  • region: UC, country: U.S.A.
    NNNNN (N: number)
    ex. 23654
  • region: UC, country: Canada
    XNXNXN (X: uppercase alphabet, N: number)
    ex. A3B6H4
  • region: JP
    NNNNNNN (N: number)
    ex. 1080075

requestToNotifyDeviceStatus v1.0+

The information of "internetTVLinkage" object is for Internet TV device (SONY Internet TV-BOX / Stick) connected to TV via HDMI/MHL.
The value of "playingInternetTV" is true when the source of Internet TV device is displayed on the connected MHL input.
The value of "showingKeyControlableGui" is true when some TV's GUI appears on screen.
It's assumed that these values are used from Internet TV Stick device to control TV and itself.

getNetworkSettings+

This API provides information about network settings.

Response includes information only about active network interface.

-  The active network interface is assigned IP address.

-  The connection setup of either Wired or Wi-Fi is possible.

   It depends on the current network settings of "Wired + (Wi-Fi Direct)" or "Wi-Fi + (Wi-Fi Direct)".

-  When the "built-in Wi-Fi" set on and the connection setup is Wi-Fi, IP address is assigned, even if it is not connected to the access point.

Guide about Request

+

name

explanation

netif

Value of "netif" is "eth0", "wlan0", "p2p0" or "". Empty means all interfaces.

Guide about Response+
  • The value of "ipAddrV6" is always empty.
  • In case TV is in power off state error code 7 (illegal state) is returned.

setPlayContent v1.0+

Guide about Request+
  • supported uri is replied by getContentList.
  • "tv:" to tune in to last TV channel.
  • As for broadcasting channels, API implementation of this version supports Digital broadcasting channels only. In other words, it cannot change to Analog channels.
  • On the other hand, the types of list that this API doesn't play are:
    • media file in USB storage, for example, mp3 music files, jpeg files or mp4 video files.
    • media file provided by DLNA server
    • IPTV media content provided by BIVL or SEN service.

  • Note
    • Smart-Core on TV lights on after calling this API. (for TV channel and REC content)
    • In ATSC region, the "index" included in the uri parameter must be set.
      If the "index" is not set, complement by getting the "index" of top priority.

getApplicationList v1.0+

Guide about Response+

  • There are 3 types of application as below. It can be fetched using the prefix of uri.

    application typeexplanation

    Preset

    means TV built-in application.
    The prefix of uri is "preset://".

    AApp

    means Downloadable application that is pre-installed or installed by user.
    The prefix of uri is "aapp://".

    Kamaji

    means Network based application, for example, XWidget or BIVL.
    The prefix of uri is "kamaji://".

  • Below is additional explanation of response.

    nameexplanation

    title

    If the application has DIC Key based name on TV; for example {TV_GUIDE_APP}, it's converted to the human-readable text in the result.

    uri

    Application URI that can be used to launch it on TV.

    icon

    URL that indicates the location of the icon.
    Icons for preset and downloadable application are located in TV inside, so the URL reaches TV on LAN.
    Icons for kamaji application may be located in a public server on Web internet.

    For example, URL for preset application as below.
    http://192.168.0.3/sony/appControl/icon/pack:%3AAxxxxx.png

    For example, URL for downloadable application as below.
    http://192.168.0.3/sony/appControl/icon/aais-icon%3Axxxxxxx.png

    For example, URL for kamaji application as below.
    http://www.XXXX.com/appli/images/lcon.png

    Icon URL for preset and downloadable applications points to TV device on internal LAN. A client can retrieve icon data by HTTP GET request to the URL.

    data

    Not returned in TV

setMultiScreenMode v1.0+

This API in TV has two particular behaviors for GTV Stick called MAGI.
First one is to terminate Widget application or Related/Text Search application on TV when one of these applications is active and the "mode" value in Request parameter is set to "single".
Second one is to change screen mode setting to full pixel when the "mode" and "internetTVMode" value in Request parameter are set to "single" and "true".
